# Plan Mode 실습 (슬 61)

## 시나리오
큰 작업은 바로 고치지 말고 계획부터 받는다. 파일 수정 없이 작업 순서와 위험을 먼저 확인한다.

## Claude 명령

```
/plan

이 폴더를 연구 자동화 템플릿으로 바꾸려면
어떤 파일을 만들고 어떤 순서로 진행할지
수정 없이 계획만 제안해줘.

위험한 점과 확인 질문도 함께 써줘.
```

Shift+Tab으로 Plan Mode를 켤 수도 있다.

## 기대 출력 (구조)
- 만들 파일·폴더 목록
- 작업 순서 (의존성 포함)
- 위험 요소 (예: 기존 파일 덮어쓰기, 외부 API 호출, 비용 발생)
- 진행 전 확인 질문

## 핵심 학습 포인트
- "수정 없이 계획만"이라는 단서가 핵심
- 위험·질문 항목을 강제하면 즉흥적 실행을 막을 수 있음
- Shift+Tab 토글로 평소 turn에서 Plan Mode로 즉시 전환 가능
