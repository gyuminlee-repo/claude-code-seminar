# 실습 1 · Markdown 노트 정리 (슬 54)

## 시나리오
주간 미팅·강의·인터뷰 후 남은 raw 메모 파일을 회의록 형식으로 재정돈한다. 결정 사항, 다음 액션, 미확정 질문을 분리해 검토 가능한 구조로 만든다.

## 입력
- `meeting-note.md` (이 repo 동봉 raw 회의록)

## Claude 명령

```
meeting-note.md를 읽고 정리해줘.

형식:
1. 오늘 논의
2. 결정 사항
3. 다음 액션
4. 미확정 질문

결정 사항은 표로 만들어줘.
```

복붙 대신 `@meeting-note.md`처럼 파일 참조를 써도 된다.

## 기대 출력 (구조)
- `1. 오늘 논의`: 토픽 bullet
- `2. 결정 사항`: Markdown 표
- `3. 다음 액션`: 담당자, 기한 포함 체크리스트
- `4. 미확정 질문`: 후속 확인 항목

## 핵심 학습 포인트
- 형식 지정만으로 raw prose에서 구조화 출력으로 전환
- 결정 사항을 표로 강제하면 재검색·재활용성이 올라감
- `@파일명` 참조 문법으로 복사·붙여넣기 부담 제거
